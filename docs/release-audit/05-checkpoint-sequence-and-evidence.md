# Mandatory checkpoint sequence and evidence

1. Establish and save this audit.
2. Reconcile every conversation instruction, complaint, correction and promised feature.
3. Inventory current package entry point, HTML, renderer wiring, menus, panels, publishing backend, validators, workflow and stale build files.
4. Reproduce and structurally fix the bar covering the staff; add layout regressions.
5. Implement approved mouse drag-out/drag-in docking, docking targets, persistence, safe dimensions and off-screen recovery.
6. Implement approved menu/tool grouping and remove obsolete or duplicate controls.
7. Integrate visible Dedicated PDF, PNG Pages and System Print controls with real actions and explicit errors.
8. Audit performance and remove observers, rescans, duplicate listeners, timers and repeated injection.
9. Run the complete regression suite without lowering thresholds; record exact totals.
10. Align version/build, package Windows installer and portable executable, validate PE files, launch the compiled app and require live renderer proof.
11. Install on Windows, isolate old versions, verify process path/build identity, inspect all layouts, menus, docking, PDF, PNG and performance.
12. Verify artifact names, sizes and SHA-256 hashes.
13. Perform adversarial self-critique.
14. Approve only when every mandatory requirement is PASS.

At every checkpoint append a dated ledger entry containing: branch, commit SHA, files changed, evidence, test results, remaining UNKNOWN/BLOCKED/FAIL/PARTIAL items, newly discovered requirements, self-critique and next checkpoint. Never delete prior entries or rewrite a failure as PASS without replacement evidence.

Evidence table columns:
Requirement ID | User instruction/source | Implementation files | Automated test | CI/runtime evidence | Human evidence | Status | Notes
